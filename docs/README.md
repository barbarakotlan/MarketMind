# Documentation

MarketMind keeps its active project docs in a few focused groups:

- `operations/`
  Release gates, smoke tests, monthly simulation, and deployment checklists.
- `product/`
  Product planning and monetization notes that still matter to current work.
- `backend/`
  Backend API and model data reference docs.

Start with the root [`README.md`](../README.md) for repository orientation, local development, and the high-level architecture.
